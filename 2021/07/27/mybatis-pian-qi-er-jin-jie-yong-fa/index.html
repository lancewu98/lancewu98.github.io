MyBatis篇其二 ——进阶用法

1.多对一映射

在mybatis的实际使用情景下，每个查询实体不一定只是八大基础类型，可能存在复杂的对象类型，这个时候就不能直接按照之前的写法处理。首先看一下多对一的情况，创建一个Student类和一个Teacher类，多个Student对应同一个Teacher。

@Data
public class Student {
    private int id;
    private String name;

    //学生需要关联一个老师
    private Teacher teacher;
}
@Data
public class Teacher {
    private int id;
    private String name;
}

由于是测试多对一的情况，所以sql语句应该是针对Student进行查询，所以我们需要对Student进行映射。创建一个StudentMapper接口和一个StudentMapper.xml文件，并将StudentMapper.xml注册到mybatis核心配置文件的中。

StudentMapper接口内容如下，我们用两种方式来实现复杂对象类型的映射：

public interface StudentMapper {

    //按照子查询方式进行多对一映射
    List getStudent();
    Teacher getTeacher(int sid);

    //针对联表查询方式进行多对一映射
    List getStudent2();
}

从上面注释可以看出多对一映射方式有两种：

1.通过子查询方式，也即对复杂对象类型进行查询后，按照数据库类型返回基础数据类型

2.通过联表查询方式，也即直接将两实体类对应的数据库表进行级联查询

无论是上述两种方式中的哪一种，都需要用到mybatis提供的一个标签：

association

关于这个标签，mybatis官方文档是这样介绍的：

关联（association）元素处理“有一个”类型的关系。 比如，在我们的示例中，一个博客有一个用户。关联结果映射和其它类型的映射工作方式差不多。 你需要指定目标属性名以及属性的javaType（很多时候 MyBatis 可以自己推断出来），在必要的情况下你还可以设置 JDBC 类型，如果你想覆盖获取结果值的过程，还可以设置类型处理器。

关联的不同之处是，你需要告诉 MyBatis 如何加载关联。MyBatis 有两种不同的方式加载关联：

1.嵌套 Select 查询：通过执行另外一个 SQL 映射语句来加载期望的复杂类型。
2.嵌套结果映射：使用嵌套的结果映射来处理连接结果的重复子集。

上面这段的两个加载关联的方式，即我们上面提到的两种多对一查询方式。

标签里面有几个属性，有共用的，有第一种方式独有的和第二种方式独有的。属性作用不多赘述，直接放xml的内容就比较清晰了。以下是StudentMapper.xml的内容：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper  namespace="com.example.dao.StudentMapper">

    <select id="getStudent" resultMap="studentTeacherMap">
        select * from student s
    </select>
    <resultMap id="studentTeacherMap" type="student">
        <result property="id" column="id"/>
        <result property="name" column="name"/>
        <!--复杂的属性需要单独处理
            对象：association
            集合：collection -->
        <association property="teacher" column="tid" javaType="Teacher" select="getTeacher"/>
    </resultMap>
    <select id="getTeacher" resultType="Teacher">
        select * from teacher where id = #&#123;sid&#125;
    </select>

    <select id="getStudent2" resultMap="studentTeacherMap2">
        select s.id sid,s.name sname,t.id tid,t.name tname
        from student s,teacher t
        where s.tid=t.id
    </select>
    <resultMap id="studentTeacherMap2" type="Student">
        <result property="id" column="sid"/>
        <result property="name" column="sname"/>
        <association property="teacher" javaType="Teacher">
            <result property="name" column="tname"/>
            <result property="id" column="tid"/>
        </association>
    </resultMap>
</mapper>

从上可以看出，这两种映射方式，都需要写的是属性是property和javaType，其中前者是本次查询中实体类里复杂对象的名称（即Student实体类的teacher变量名），后者是这个名称对应java类型（即Teacher实体类，由于存在别名所以可以不写全限定名）。

如果是子查询方式实现映射，需要额外在标签里书写的属性是column和select，前者用于将子查询的结果和当前查询的实体类在数据库内的关联字段进行映射（本例里面即将子查询查出的teacher对应到student表的tid字段），后者即标识用到的是哪个子查询。

如果是联表查询方式实现映射，则需要在标签内书写标签，用于将这个复杂对象类型的属性和本次查询出的字段映射起来。

比较推荐用第二种方式进行多对一映射，即针对联表查询方式进行映射。

2.一对多映射

数据库表是不需要变动的，实体类需要稍微修改一下。定义一个Student2类和一个Teacher2类。

@Data
public class Student2 {
    private int id;
    private String name;

    private int tid;
}
@Data
public class Teacher2 {
    private int id;
    private String name;

    //一个老师对应多个学生
    private List students;
}

从实体类可以看出，一对多和多对一的区别，在于引入了集合。对于集合类型内存在复杂对象，mybatis是如何进行映射的呢？

与多对一一样，创建一下TeacherMapper接口和TeacherMapper.xml，并把TeacherMapper.xml作为注册到mybatis核心配置文件的标签里面:

public interface TeacherMapper {

    //针对联表查询方式进行一对多映射(比较推荐使用)
    Teacher2 getTeacher2(@Param("tid") int id);

    //按照子查询方式进行一对多映射
    Teacher2 getTeacher2_2(@Param("tid") int id);
    Student2 getStudent2(int tid);

}

同样地，有两种方式进行一对多的映射，这次需要用到的标签是：

collection

从名称来看，不难看出这个标签是用来处理集合类的，这个标签和上面的标签非常类似，主要的区别是多出了一个“ofType”的属性。这个属性其实就是填写集合类的泛型的。

TeacherMapper.xml内容如下：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.dao.TeacherMapper">

    <select id="getTeacher2" resultMap="teacher2Student2Map">
        select t.id tid,t.name tname,s.id sid,s.name sname 
        from teacher t,student s 
        where t.id=s.tid and t.id = #&#123;tid&#125;
    </select>
    <resultMap id="teacher2Student2Map" type="Teacher2">
        <result property="id" column="tid"/>
        <result property="name" column="tname"/>
        <!--复杂的属性需要单独处理，对象：association 集合：collection
        javaType=“ ” 指定属性的类型
        集合中的泛型信息，使用ofType获取-->
        <collection property="students" ofType="Student2">
            <result property="id" column="sid"/>
            <result property="name" column="sname"/>
            <result property="tid" column="tid"/>
        </collection>
    </resultMap>

    <select id="getTeacher2_2" resultMap="teacher2Student2Map2">
        select * from teacher t where t.id = #&#123;tid&#125;
    </select>
    <resultMap id="teacher2Student2Map2" type="Teacher2">
        <collection property="students" javaType="ArrayList" ofType="Student2" select="getStudent2" column="id"/>
    </resultMap>
    <select id="getStudent2" resultType="Student2">
        select * from student where tid = #&#123;tid&#125;
    </select>


</mapper>

内容一目了然，不做赘述。

3.动态SQL

关于动态SQL，mybatis官方文档是这样介绍的：

动态 SQL 是 MyBatis 的强大特性之一。如果你使用过 JDBC 或其它类似的框架，你应该能理解根据不同条件拼接 SQL 语句有多痛苦，例如拼接时要确保不能忘记添加必要的空格，还要注意去掉列表最后一个列名的逗号。利用动态 SQL，可以彻底摆脱这种痛苦。

使用动态 SQL 并非一件易事，但借助可用于任何 SQL 映射语句中的强大的动态 SQL 语言，MyBatis 显著地提升了这一特性的易用性。

动态sql标签如下：

  • if
  • choose (when, otherwise)
  • trim (where, set)
  • foreach

编写一个实体类Blog来测试一下这些标签的功能：

@Data
@AllArgsConstructor
public class Blog {
    private String id;
    private String title;
    private String author;
    private Date createTime;
    private int views;
}

Blog对应的数据库表内容和结构如下：


同样的，BlogMapper接口和BlogMapper.xml，并将这个xml注册到核心配置文件。

public interface BlogMapper {

    List queryBlogIf(Map map);

    List queryBlogChoose(Map map);

    int updateBlog(Map map);

    List queryBlogForeach(Map map);
}

if

标签常用于where内，当条件成立时会拼接标签内的语句，不成立则不会

 <select id="queryBlogIf" parameterType="map" resultType="blog">
        select * from blog where 1=1 
        <if test="title != null">
           and title = #&#123;title&#125;
        </if>
        <if test="author != null">
           and author = #&#123;author&#125;
        </if>
    </select>

where

标签可以用于解决上面标签的例子里，需要拼接一个“1=1”来防止where和and直接相连的情况，会自动去掉没有用处的and。

 <select id="queryBlogIf" parameterType="map" resultType="blog">
        select * from blog
        -- where只有有至少一条子语句才会拼接到sql里，且会自动去掉无用的and
        <where>
             <if test="title != null">
                  title = #&#123;title&#125;
                   </if>
            <if test="author != null">
                and author = #&#123;author&#125;
                </if>
        </where>
    </select>

如上，就算title为空，author不为空，where标签在拼接时也会把多余的and去掉防止报错。

上面这一段还可以引入一个优化一下，标签和标签配合可以抽取公共语句以复用：

 <!--sql标签可以抽取部分sql代码，然后在需要使用的地方用include标签引用-->
    <sql id="if-title-author">
        <if test="title != null">
            title = #&#123;title&#125;
        </if>
        <if test="author != null">
            and author = #&#123;author&#125;
        </if>
    </sql>

 <select id="queryBlogIf" parameterType="map" resultType="blog">
        select * from blog
        <where>
            <include refid="if-title-author"/>
        </where>
    </select>

choose

标签一般和，搭配使用，效果类似于java内的switch。

 <select id="queryBlogChoose" parameterType="map" resultType="blog">
        select * from blog
        <where>
            -- choose有点像java里的switch
            <choose>
                <when test="title != null">title = #&#123;title&#125;</when>
                <when test="author != null">and author = #&#123;author&#125;</when>
                <otherwise>and views = #&#123;views&#125;</otherwise>
            </choose>
        </where>
    </select>

相当于switch，相当于case，相当于default。

set

就是update语句那个set，作用是会自动去掉多余的逗号，这一点类似where自动去掉多余的and。

<update id="updateBlog" parameterType="map">
        update blog
        --set会自动去掉不必要的逗号
        <set>
            <if test="title != null">title = #&#123;title&#125;,</if>
            <if test="author != null">author = #&#123;author&#125;</if>
        </set>
        where id = #&#123;id&#125;
    </update>

foreach

这个标签用于遍历，使用场景如where语句的in，可以循环取数拼接。

<!--在map中放一个key为“ids”的集合，然后使得foreach里面可以使用-->
    <select id="queryBlogForeach" parameterType="map" resultType="blog">
        select * from blog
        <where>
            <foreach collection="ids" item="id" open=" (" close=")" separator="or">
                -- 花括号里的id即item里的
                id = #&#123;id&#125;
            </foreach>
        </where>
    </select>

以上遍历执行后会拼接成类似（id= 1 or id=2 pr id=3 or id=4）的样式

4.缓存

分为一级缓存和二级缓存

一级缓存

  • 存在于sqlSession中，sqlSession关闭则缓存关闭，一级缓存默认开启
  • select操作的结果会被缓存
  • insert update delete操作会刷新缓存(即重新查库把数据写入缓存)
  • 缓存会使用最近最少使用算法（LRU, Least Recently Used）算法来清除不需要的缓存
  • 缓存不会定时进行刷新（也就是说，没有刷新间隔）
  • 缓存会保存列表或对象（无论查询方法返回哪种）的 1024 个引用
  • 缓存会被视为读/写缓存，这意味着获取到的对象并不是共享的，可以安全地被调用者修改，而不干扰其他调用者或线程所做的潜在修改

一级缓存默认开启，一个sqlSession里面拥有一个，不同sqlSession之间无法互相读取缓存数据。如果有这种需求怎么处理呢？那就需要用到二级缓存了。

二级缓存

在核心配置文件mybatis-config.xml中，配置里面显式地开启全局缓存：

    <!--配置-->
    <settings>
        <!--显式地开启全局缓存-->
        <setting name="cacheEnabled" value="true"/>
    </settings>

在BlogMapper.xml里面，加上以下标签就可以开启二级缓存：

<cache/>
  • 开启二级缓存后，一级缓存提交或关闭时，会把结果写入二级缓存，使得其他会话可以读取到缓存的数据
  • 如果这个标签没有配置策略，则需要把实体类使用序列化接口
  • 二级缓存只对这个mapper有效
  • 查询先走二级缓存，再一级，再数据库
  • 可以用type属性绑定自定义缓存，如ehcache等
